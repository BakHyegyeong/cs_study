# 멀티프로세싱, 멀티프로그래밍, 멀티스레딩, 멀티태스킹의 차이점

### 멀티프로세싱
<img src="https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdn%2Feis2uX%2FbtrHWvQxQDl%2FLy3pakpsMIkv6vu8hzMIp1%2Fimg.png" width="70%">

- 하나의 작업을 두 개 이상의 프로세서가 동시에 처리하는 것. 물리적으로 여러 프로세서가 있어 각 프로세스가 별도 CPU에서 동시에 실행됩니다.
- 등장 배경
    - 하나의 프로세서로는 성능 향상이 어려워짐에 따라 여러 개의 프로세서를 사용하여 성능을 향상시키기 위해 등장
- 특징
  - 병렬 처리로 성능 향상

###### 프로세서 : CPU같은 하드웨어 장치
###### 프로세스 : 실행 중인 프로그램

### 멀티프로그래밍
<img src="https://velog.velcdn.com/images%2Fmaketheworldwise%2Fpost%2Fe12ba22d-3504-4ce9-b888-c0383c6c63f8%2Fimage.png" width="70%">

- 하나의 CPU가 여러 프로그램을 번갈아 가며 실행하는 방식
- 등장 배경
  - 초기 컴퓨터는 하나의 CPU가 하나의 프로세스만 처리하도록 설계되었다.
  - 하나의 프로세스를 처리하는 과정에서 프로세스의 입출력 처리를 기다리는 비효율적인 상황이 발생하면서 자원 낭비가 발생하므로 이를 방지하기 위해 등장
- 특징
  - 대기 시간 최소화
  - CPU 사용률 개선(CPU가 놀지 않고 일을 처리)
  - 자원 활용도 향상(자원 낭비 방지)

### 멀티스레딩
<img src="https://velog.velcdn.com/images/hkh1213/post/fa5ab1a8-0e06-4ccf-be2e-030f5d0ddc0b/image.png" width="70%">

- 하나의 프로세스를 다수의 스레드로 구분하여 자원을 공유하고 자원 관리를 통해 수행 능력을 향상시키는 것
- 등장 배경
  - 스레드를 생성하는 비용이 더 저렴하기 때문에 프로세스에 다수의 스레드를 생성하여 병렬처리하는 것이 더 효율적이다.
  - 프로세스는 독립적인 메모리 구조를 갖고 있지만 스레드는 스택 영역을 제외한 데이터, 힙 영역을 서로 공유하므로 효율적이다.
- 특징
  - 자원 소비 최소화(메모리 공유)
  - 응답성 향상(스레드 간 통신을 통해 빠른 처리 가능)

### 멀티태스킹
<img src="https://s3.ap-northeast-2.amazonaws.com/static.gracieuxyh.dev/os/multi-tasking.png" width="70%">

- 다수의 작업(Task)을 동시에 실행하는 운영체제의 기능. 컨텍스트 스위칭을 통해 여러 개의 task를 번갈아 가며 수행되도록 한다.
- 특징
  - 사용자는 동시에 여러 task가 수행되고 있다고 느끼게 된다.(사용자 경험 향상)
  - 효율성 개선

### 멀티프로세싱과 멀티스레딩의 차이점
  - 멀티 프로세싱은 하나의 작업을 두 개 이상의 CPU가 동시에 처리하는 것
  - 멀티 스레딩은 하나의 프로세스내에서 스레드를 생성하여 병렬처리하는 것

### 멀티프로그래밍과 멀티태스킹의 차이점
  - 멀티 프로그래밍은 자원의 낭비를 막는 것을 목적으로 하며 멀티 태스킹을 구현하는 스케줄링의 한 방식이다.
  - 멀티 태스킹은 동작이라는 관점에서 차이가 있다.
